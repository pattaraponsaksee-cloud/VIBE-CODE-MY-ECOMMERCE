# 🧠 NEUROVIBE - เอกสารสรุปการทำงานของระบบ (Project Architecture & Codebase Summary)

เอกสารฉบับนี้จัดทำขึ้นเพื่ออธิบายโครงสร้างและการทำงานเชิงลึกของโปรเจกต์ **NEUROVIBE** ทั้งในส่วนของหน้าบ้าน (Frontend), หลังบ้าน (Backend), ฐานข้อมูล (Database) และการเชื่อมโยงข้อมูลเข้าด้วยกัน เพื่อให้นำข้อมูลนี้ไปใช้งานใน **NotebookLM** ได้อย่างมีประสิทธิภาพสูงสุด

---

## 1. ภาพรวมของโปรเจกต์ (Project Overview)
**NEUROVIBE** คือเว็บแอปพลิเคชันรูปแบบร้านค้าออนไลน์ (E-Commerce) ที่จำหน่ายสินค้าแบบไฮบริด (Hybrid Store):
1. **สินค้าประเภทยากายภาพ (Physical Product):** วิตามินเสริมบำรุงสมองและสมาธิ (Nootropic Supplements) จัดส่งทางไปรษณีย์
2. **สินค้าประเภทดิจิทัล (Digital Product):** สมุดแพลนเนอร์วางแผนตารางชีวิตดิจิทัล (Digital Planner) ส่งลิงก์ดาวน์โหลดทางอีเมลทันที

---

## 2. ส่วนหน้าบ้าน (Frontend Architecture)
ตั้งอยู่ในโฟลเดอร์ `VIBE-CODE-MY-ECOMMERCE/app/` ทำงานแบบไคลเอนต์ไซด์ (Client-Side Rendering) ประกอบด้วย 3 ไฟล์หลัก:

* **index.html (โครงสร้างและการจัดวาง):**
  * โครงสร้างแบบ Semantic HTML แบ่งเป็น Header Navigation, Hero Banner, Product Section, Reviews Carousel, Cart Drawer, Checkout Modal, และ Footer
  * นำเข้าสไตล์ภายนอกผ่าน CDN เช่น ไอคอนของ Remix Icon และฟอนต์จาก Google Fonts (Outfit & Inter)

* **style.css (ระบบดีไซน์และธีมพรีเมียม):**
  * ดีไซน์แนว **Biotech / Cyber-Wellness** ที่ดูทันสมัยและสะอาดตา
  * ใช้ **Dark Mode** โดยใช้สีพื้นหลังน้ำเงินเข้มจัด (`#070B13`) ตัดกับสีเรืองแสงของ Cyan (`#00F2FE`) และ Purple (`#9B51E0`)
  * ตกแต่งด้วยแผงการ์ดแบบโปร่งแสงสะท้อนแสงกระจก (Glassmorphism) และแอนิเมชันลูกกลมเรืองแสงหมุนช้า ๆ ด้านหลัง (Pulsing Background Orbs)
  * มีสไตล์การตอบสนองที่ยืดหยุ่น (Responsive Design) ปรับเปลี่ยนหน้าตาตามความกว้างหน้าจอมือถือและเดสก์ท็อปโดยอัตโนมัติ

* **app.js (ตรรกะระบบร้านค้าทั้งหมด):**
  * **Database จำลอง (Array):** เก็บอาร์เรย์ข้อมูลสินค้า (`PRODUCTS`) และรีวิวระดับ 5 ดาว (`REVIEWS`) 
  * **ระบบตะกร้าสินค้า (Shopping Cart):** จัดการสถานะการเพิ่มสินค้า การลดจำนวนชิ้น การคำนวณราคาสินค้ารวม และค่าจัดส่ง (บวกเพิ่ม 50 บาทเฉพาะออเดอร์ที่มีวิตามินแบบกายภาพ ส่วนออเดอร์ที่มีเฉพาะดิจิทัลแพลนเนอร์จัดส่งฟรี 0 บาท)
  * **ฟอร์มชำระเงินอัจฉริยะ (Checkout Logic):** ปรับเปลี่ยนช่องกรอกข้อมูลตามประเภทสินค้าที่สั่งซื้อ เช่น หากไม่มีวิตามินทางกายภาพในตะกร้า จะซ่อนกล่องกรอก "ที่อยู่สำหรับจัดส่ง" ทันที เพื่อความรวดเร็วในการสั่งซื้อสินค้าดิจิทัล
  * **หน้าจอชำระเงินสำเร็จ:** ล้างค่าในตะกร้าสินค้า และสร้าง **ลิงก์ดาวน์โหลดไฟล์ PDF** ของแพลนเนอร์ที่สั่งซื้อให้คลิกดาวน์โหลดได้ทันที

## 3. ส่วนหลังบ้าน (Backend Architecture)
ตั้งอยู่ในโฟลเดอร์ `VIBE-CODE-MY-ECOMMERCE/app/api/` รันด้วยภาษา JavaScript บนสภาพแวดล้อม Node.js มีไฟล์หลักคือ:

* **server.js (ตัวประมวลผลเซิร์ฟเวอร์):**
  * รันเว็บเซิร์ฟเวอร์ด้วยเฟรมเวิร์ก **Express.js** ที่พอร์ต `3000`
  * ใช้เครื่องมือ **EJS (Embedded JavaScript Templates)** เพื่อแสดงผลหน้าเว็บฝั่งผู้ดูแลระบบ (Admin Dashboard) เช่น หน้ารายการออเดอร์, หน้ารายชื่อลูกค้า และหน้ารายละเอียดออเดอร์
  * **การจัดการเส้นทาง (Routing):**
    * `/` (Dashboard): รายงานสถิติจำนวนลูกค้า, สินค้า, รายการออเดอร์ และคำนวณยอดรายได้สุทธิ
    * `/products`: ค้นหา ฟิลเตอร์ และจัดแสดงรายละเอียดของสินค้า
    * `/users`: แสดงบัญชีรายชื่อลูกค้าทั้งหมดในระบบ
    * `/orders`: คัดกรอง ค้นหาออเดอร์ล่าสุด และสั่งอัปเดตสถานะการจัดส่ง ตลอดจนสามารถกำหนดช่างเทคนิคผู้ดูแลการติดตั้งได้ผ่านคำร้องขอแบบ POST

## 4. ระบบฐานข้อมูล (Database & ER Diagram)
ตัวระบบหลังบ้านเชื่อมต่อเข้ากับฐานข้อมูล **MongoDB** (ใช้บริการคลาวด์ MongoDB Atlas) เก็บข้อมูลในรูปแบบ Document-based โดยมีความสัมพันธ์ของข้อมูลแสดงผ่าน **ER Diagram (er_diagram.md)** ดังนี้:

* **USERS (เก็บข้อมูลลูกค้า):**
  * สัมพันธ์กับตาราง **ORDERS** ในรูปแบบ **1-to-Many (หนึ่งคนสั่งได้หลายออเดอร์)**
* **ORDERS (เก็บยอดชำระและประวัติสั่งซื้อ):**
  * สัมพันธ์กับตาราง **ORDER_ITEMS** ในรูปแบบ **1-to-Many (หนึ่งใบสั่งซื้อมีสินค้าได้หลายชิ้น)**
* **PRODUCTS (รายการสินค้าทั้งหมด):**
  * สัมพันธ์กับตาราง **ORDER_ITEMS** ในรูปแบบ **1-to-Many (สินค้าหนึ่งชิ้นสามารถถูกขายได้ในหลายรายการสั่งซื้อ)**
  * สัมพันธ์กับตาราง **REVIEWS** ในรูปแบบ **1-to-Many (สินค้าหนึ่งชิ้นรับข้อความรีวิวประเมินจากลูกค้าได้หลายรายการ)**

## 5. การทำงานร่วมกันของทั้งระบบ (System Flow)
1. **ลูกค้าชมหน้าเว็บ (Frontend):** ลูกค้าเลือกดูวิตามินบำรุงสมองหรือดิจิทัลแพลนเนอร์บนหน้าเว็บ `index.html` ที่ออกแบบอย่างสวยงาม
2. **ทำรายการซื้อสินค้า:** กดเพิ่มสินค้าลงตะกร้า ระบบ JavaScript (`app.js`) คำนวณราคารวมและค่าจัดส่งแบบไดนามิก จากนั้นลูกค้ากดปุ่ม Checkout กรอกชื่อ อีเมล และที่อยู่ (ถ้ามี)
3. **บันทึกรายการ (หลังบ้าน & ฐานข้อมูล):** ข้อมูลใบสั่งซื้อจะถูกส่งมายังหลังบ้านผ่าน API (รันโดย `server.js`) ซึ่งจะเข้าไปตรวจสอบสิทธิ์และทำการบันทึกข้อมูลลูกค้า และบันทึกออเดอร์ใหม่ลงฐานข้อมูล **MongoDB**
4. **จัดส่ง & การจัดการออเดอร์:** หลังบ้านจะตรวจสอบประเภทสินค้า หากเป็นสินค้าดิจิทัล ระบบจะส่งไฟล์ผ่านอีเมล หากเป็นวิตามินจะส่งที่อยู่ให้ทีมคลังจัดส่งต่อไป โดยผู้ดูแลระบบสามารถตรวจสอบและปรับสถานะออเดอร์ผ่านหน้าต่าง **Admin Dashboard** ได้ตลอดเวลา
